# VLM-ARB Cloud Setup Checklist

**Step-by-step checklist for team lead to set up cloud framework**

---

## Total Time: ~30 minutes

### Phase 1: Firebase Project (10 min)

- [ ] Go to https://console.firebase.google.com
- [ ] Click "Create a project"
- [ ] Project name: `vlm-arb-{semester}` (e.g., `vlm-arb-spring2026`)
- [ ] Uncheck "Enable Google Analytics"
- [ ] Click "Create project"

### Phase 2: Firestore Setup (5 min)

- [ ] Firebase Console → Firestore Database (left menu)
- [ ] Click "Create database"
- [ ] Mode: **Production**
- [ ] Region: `us-central1`
- [ ] Click "Create"
- [ ] Wait for Firestore to initialize (~1 min)

### Phase 3: Cloud Storage Setup (5 min)

- [ ] Firebase Console → Storage (left menu)
- [ ] Click "Get Started"
- [ ] Mode: **Production**
- [ ] Region: `us-central1`
- [ ] Click "Continue"
- [ ] Bucket name: `{project-id}.appspot.com` (auto-populated)
- [ ] Wait for Storage to initialize (~1 min)

### Phase 4: Service Account Credentials (3 min)

- [ ] Firebase Console → Project Settings (⚙️ icon, top-right)
- [ ] Tab: "Service Accounts"
- [ ] Language: "Python" (recommended)
- [ ] Click "Generate New Private Key"
- [ ] Save JSON file as: `serviceAccountKey.json`
- [ ] ⚠️ **KEEP SECURE** - Contains credentials

### Phase 5: Security Rules (3 min)

#### Firestore Rules:

- [ ] Firebase Console → Firestore → Rules tab
- [ ] Delete all existing rules
- [ ] Paste contents of [firestore-rules.txt](firestore-rules.txt)
- [ ] Click "Publish"

#### Cloud Storage Rules:

- [ ] Firebase Console → Storage → Rules tab
- [ ] Delete all existing rules
- [ ] Paste contents of [storage-rules.txt](storage-rules.txt)
- [ ] Click "Publish"

### Phase 6: Google Drive Setup (2 min)

- [ ] Open Google Drive
- [ ] Create new folder: `VLM-ARB-Team`
- [ ] Create subfolder: `secrets` (inside VLM-ARB-Team)
- [ ] Create subfolder: `notebooks` (inside VLM-ARB-Team)

### Phase 7: Upload Credentials (1 min)

- [ ] Upload `serviceAccountKey.json` to `Drive/VLM-ARB-Team/secrets/`
- [ ] Share VLM-ARB-Team folder with team (click Share button)
- [ ] Set permissions: **Viewer** (they can read, not modify)

### Phase 8: Configure Project ID (1 min)

- [ ] Open [config.yaml](config.yaml)
- [ ] Find line: `project_id: "YOUR_PROJECT_ID"`
- [ ] Replace `YOUR_PROJECT_ID` with actual Firebase project ID
- [ ] Save and upload to Drive/VLM-ARB-Team/
- [ ] (Or team can edit locally in Colab)

### Phase 9: Copy Notebooks (1 min)

- [ ] Download 3 notebooks from repo:
  - `Notebook_1_Generate_Datasets.ipynb`
  - `Notebook_2_Run_Model_Evaluations.ipynb`
  - `Notebook_3_Generate_Reports.ipynb`
- [ ] Upload to `Drive/VLM-ARB-Team/notebooks/`
- [ ] Share with team (inherit from parent folder)

### Phase 10: Quick Test (2 min)

- [ ] Open Notebook 1 in Colab (File → Open from Drive)
- [ ] Run Cell 1-2 to verify:
  - [ ] Dependencies install ✅
  - [ ] Firebase auth succeeds ✅
  - [ ] Credentials load ✅
- [ ] If all pass → **Setup complete!** ✅

## Common Mistakes to Avoid

❌ **Don't**:

1. Share `serviceAccountKey.json` in Slack/email
   - Only share Drive folder link via email

2. Use "Test mode" for Security Rules
   - Use "Production" mode with explicit rules

3. Forget to Publish rules
   - Must click "Publish" after editing rules

4. Delete old evaluation results
   - Security rules prevent `delete` on `results/` collection

5. Store large files (>1GB) in Drive instead of Cloud Storage
   - Use Cloud Storage for datasets and reports

6. Use same Firebase project for personal experiments
   - Create dedicated project for team collaboration

✅ **Do**:

1. Store credentials in Drive's `secrets/` subfolder
2. Test in Colab Cell 2 before running full notebooks
3. Use unique run IDs (auto-generated by notebooks)
4. Monitor Firestore quotas monthly
5. Keep config.yaml updated in Drive
6. Document dataset versions in team chat
